## 开机引导方式 → 磁盘存储数据的结构

> **注意：这两者的对应关系！**

| **开机引导方式**            | **磁盘存储数据的结构** |
| --------------------------- | ---------------------- |
| legacy 引导（适合老旧主板） | MBR 分区               |
| UEFI 引导                   | GPT 分区               |

# 步骤

<a name="LMfFy"></a>

## 1、制作 PE 启动盘

> **微PE：**[**http://www.wepe.com.cn/download.html**](http://www.wepe.com.cn/download.html)

1. 准备足够大小的U盘（至少8G）
2. 下载对应的版本。
3. 根据提示格式化U盘。
4. 在空分区中准备重装的系统镜像、工具……

> 注意：**U盘会分为两个区**
>
> 1. **EFI 分区**：PE本身所在的区。不要动！
> 2. **空白分区**：可当作U盘使用即可，可以放置 windows 镜像文件等……

<a name="Z2RXb"></a>

## 2、BIOS 设置

1. 备份好资料。
2. 重启电脑，进入 BIOS 界面。
3. 设置U盘为启动项，保存设置重启。 <a name="MaZ1W"></a>

## 3、开始安装系统

4. 进入 PE 界面
5. 使用分区工具（如：DiskGenius）进行磁盘分区

> 注意：
>
> - 格式化磁盘，**不要格式化U盘！**
> - 固态硬盘设置 4k 对齐。

# 分区 - 踩坑

<a name="dtKcE"></a>

## 并未启动 U 盘，直接启动原来的系统

将U盘设为第一启动项，或者在启动时，选择启动U盘 <a name="KV5dO"></a>

## 弹框提示：Windows无法安装到这个磁盘。选中的磁盘具有MER分区表。在EFI 系统上，Windows只能安装到GFT磁盘。

> **原因：**这个报错的原因就是 **开机引导 **和 **磁盘数据结构 **导致的！

<a name="tP2b4"></a>

## 解决方案1

老电脑：legacy 引导启动，匹配 MBR 磁盘 <a name="nPMwf"></a>

## 解决方案2

1. 进入 BIOS 更改引导方式为 UEFI（通常在 BOOT 选项里面修改）
2. <br />
   1. **第三方工具**：如分区工具，将 MBR 切换成 GPT，用来匹配UEFI引导
   2. **命令行的方式**：将 MBR 切换成 GPT，用来匹配UEFI引导
   - `Shift + F10` 打开命令提示符
   - 输入： `diskpart` 
      - 回车
   - 输入： `list disk` 
      - 回车。将会列出电脑的所有磁盘
   - 输入： `select disk 0` 
      - **注意：此时的 **`**0**`** 是代指的电脑磁盘，注意区分U盘和电脑磁盘**
      - 回车。选中电脑磁盘
   - 输入： `clean` 
      - 回车。清空选的磁盘的全部分区
   - `convert gpt` 
      - 回车。此时磁盘就切换成 GPT 格式了
      - 同理：如果是 `convert MBR` ，磁盘就切换成 MBR
   - 输入两次 `exit` ，回车即可退出
