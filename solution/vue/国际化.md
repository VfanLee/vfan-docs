# 国际化

> 参见：[Vue I18n](https://github.com/intlify/vue-i18n-next#readme)

## 逻辑分析

1. 定义国际化文本
2. 通过不同地区语言展示不同文本

## 实现示例

## 安装依赖

```sh
# Vue 3 使用 V9+
npm i vue-i18n@9
```

## locale 配置

参考如下目录结构创建对应文件。

```
- locale      # i18n 文件夹
  - index.js  # i18n 配置文件
  - zh.json   # 中文包
  - en.json   # 英文包
- main.js     # 项目入口
```

### vue-i18n 入口文件

```js
import { createI18n } from 'vue-i18n'
import zh from './zh.json'
import en from './en.json'
// ...

const messages = {
  zh,
  en,
  // ...
}

const i18n = createI18n({
  legacy: false,        // you must set `false`, to use Composition API
  locale: 'zh',         // set locale
  fallbackLocale: 'en', // set fallback locale
  messages,             // set locale messages
})

export default i18n
```

### 引入 i18n 配置文件

```js
import { createApp } from 'vue'

import i18n from './locale'

createApp(App)
  .use(i18n)
  .mount('#app')
```

## 切换语言

```js
// 方案1
import { useI18n } from 'vue-i18n'

const i18n = useI18n()

// when vue-i18n is being used with legacy: false, note that i18n.global.locale is a ref, so we must set it via .value:
i18n.locale.value = 'en'

// otherwise - when using legacy: true, we set it like this:
i18n.locale = 'en'
```

```js
// 方案2
import i18nLocale from '@/locale'

// when vue-i18n is being used with legacy: false, note that i18n.global.locale is a ref, so we must set it via .value:
i18nLocale.global.locale.value = 'en'

// otherwise - when using legacy: true, we set it like this:
i18nLocale.global.locale = 'en'
```

对于一些单独的 js 文件，可能会有响应式丢失，导致更改语言后字段没有更新的情况，可以使用如下方案解决：

1. 传递翻译字段，在响应式页面中再通过 `$t()`、`t()` 来处理。
2. 使用 `location.reload()` 来刷新页面来重新加载该 js 文件。

## 使用示例

### 组件内使用国际化字段

```html
<script>
import { useI18n } from 'vue-i18n'

const $i18n = useI18n()

// 使用国际化语言字段
console.log($i18n.t('test1'))
console.log($i18n.t('test2'))
console.log($i18n.t('test3'))
</script>

<template>
  <!-- 当前语言 -->
  <div>{{ $i18n.locale }}</div>

  <!-- 使用国际化语言字段 -->
  <div>{{ $t('test1') }}</div>
  <div>{{ $t('test2') }}</div>
  <div>{{ $t('test3') }}</div>
</template>
```

### 组件外使用国际化字段

```js
import i18nLocale from '@/locale'

const t = i18nLocale.global.t

// 使用国际化语言字段
console.log(t('test1'))
console.log(t('test2'))
console.log(t('test3'))
```
