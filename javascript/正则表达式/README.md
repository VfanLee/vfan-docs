# 正则表达式

> 在线正则：https://regex101.com/

## 1. 模式修正符

- `g`；全局模式，匹配全部可匹配结果。
- `m`：多行模式，该模式下 `^` 和 `$` 可以分别匹配行首和行尾。
- `i`：不区分大小写。
- `s`：单行模式，该模式下能匹配任意字符，包括换行符。

```js
/foo/gmi
```

## 2. 元字符

> 当匹配字符与关键符冲突时，使用 `\` 转义符来进行修饰。

- `.`：匹配除换行以外的任意字符。
- `\s`：匹配空格、制表符和换行符等空白字符。
- `\S`：匹配除空格、制表符和换行符以外的字符。
- `\d`：匹配数字，等效于 `[0-9]`。
- `\D`：匹配非数字，等效于 `[^0-9]`。
- `\w`：匹配任何字母、数字、下划线 `_`，等效于 `[a-zA-Z0-9_]`。
- `\W`：匹配非字母、数字、下划线 `_`，等效于 `[^a-zA-Z0-9_]`。

## 3. 数量限制

- `a{3}`：3个a。
- `a{3,9}`：3-9个a。
- `a{3,}`：至少3个a。
- `a{,9}`：最多9个a。
- `a?`：0个或者1个a。尽可能匹配多的，贪婪。
- `a+`：1个或者多个a。尽可能匹配多的，贪婪。
- `a*`：0个或者多个a。尽可能匹配多的，贪婪。
- `a??`、`a+?`、`a*?`：尽可能匹配少的，懒惰。

## 4. 字符边界约束

- `^`：限定起始边界。
- `$`：限定结束边界。

```js
// 匹配 a-z 字符，且长度限定为 3-6 位。
/^[a-z]{3,9}$/g
```

## 5. `[]` 原子表

```js
// 匹配 1、2、3 中的任意一个字符
/[123]/g
```

### 5.1. 排除匹配

注意：原子表中的 `^`（非） 和 字符边界约束中的 `^`（以...开头） 含义并不相同！

```js
// 匹配除 1、2、3 以外的任何字符
/[^123456]/g
```

### 5.2. 区间匹配

```js
// 匹配 0-9
/[0-9]/g

// 匹配 26 个小写字母
/[a-z]/g
```

## 6. `()` 原子组

- 原子组是可以嵌套的
- `\1` 表示匹配第一个原子组的字符串。

```js
let temp = `
  <h1>foo</h1>
  <h1>foo</h2>
  <h2>bar</h2>
  <h3></h3>
`
// 匹配符合 h1-h6 标签
/<(h[1-6])>(.*)<\/\1>/g
```

### 不记录组

```js
/(?:...)/
```

## 7. `|` 或选择符

```js
// 匹配 foo12 或者 34 或者 56
/foo12|34|56/g

// 匹配 foo12 或者 foo34 或者 foo56
/foo(12|34|56)/g
```
