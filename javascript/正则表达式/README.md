# 正则表达式

> 在线正则：https://regex101.com/

## 模式修正符

- `g`；全局模式，匹配全部可匹配结果。
- `m`：多行模式，该模式下 `^` 和 `$` 可以分别匹配行首和行尾。
- `i`：不区分大小写。
- `s`：单行模式，该模式下能匹配任意字符，包括换行符。

```js
/foo/gmi
```

## 元字符

> 当匹配字符与关键符冲突时，使用 `\` 转义符来进行修饰。

- `.`：匹配除换行以外的任意字符。
- `\s`：匹配空格、制表符和换行符等空白字符。
- `\S`：匹配除空格、制表符和换行符以外的字符。
- `\d`：匹配数字，等效于 `[0-9]`。
- `\D`：匹配非数字，等效于 `[^0-9]`。
- `\w`：匹配任何字母、数字、下划线 `_`，等效于 `[a-zA-Z0-9_]`。
- `\W`：匹配非字母、数字、下划线 `_`，等效于 `[^a-zA-Z0-9_]`。

## 数量限制

- `a{3}`：3个a。
- `a{3,9}`：3-9个a。
- `a{3,}`：至少3个a。
- `a{,9}`：最多9个a。
- `a?`：0个或者1个a。
- `a*`：0个或者多个a。
- `a+`：1个或者多个a。

## 字符边界约束

- `^`：限定起始边界。
- `$`：限定结束边界。

```js
// 匹配 a-z 字符，且长度限定为 3-6 位。
/^[a-z]{3,9}$/
```

## | 选择符

```js
// 匹配 12 或者 34 或者 56
/12|34|56/
```

## [] 原子表

> 注意：原子表中的 `^`（非） 和 字符边界约束中的 `^`（以...开头） 含义并不相同！

```js
// 匹配 123456 中的任意一个字符
/[123456]/

// 匹配除 1、2、3、4、5、6 以外的字符
/[^123456]/
```

## () 原子组

```js
// 匹配 aa 或者 bb 或者 cc
/(12|34|56)/
```
