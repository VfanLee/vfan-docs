# 认证

## 登录

核心：

1. 成功登录后，缓存用户相关信息，如：token、userinfo。
2. 跳转成功登录页面。

### SSO 单点登录

SSO 角色：

- `IdP` 即身份提供方，是一种存储和管理数字身份的服务，可以让用户使用一个账号和密码来登录多个应用或网站，实现单点登录 (SSO) 的功能。
- `SP` 即服务提供方，指提供应用或网站服务的一方，需要验证用户的身份。

SSO 协议：

`IdP` 和 `SP` 之间通常使用一种协议来交换身份信息，如：

- `SAML`：即 Security Assertion Markup Language，一种基于 XML 的标准，用于在 IdP 和 SP 之间传递身份断言和认证信息。
- `OAuth`：即 Open Authorization，一种基于 HTTP 的授权协议，用于让用户授权第三方应用或网站访问他们在 IdP 中存储的受保护的资源，如个人信息、照片、视频等。
- `OpenID Connect`：一种基于 OAuth 2.0 的身份验证协议，用于在 IdP 和 SP 之间传递用户的身份信息和其他属性，如姓名、邮箱、头像等。

## 退出登录

退出登录可以分为两种：

1. 主动退出：用户手动点击退出登录按钮。
2. 被动退出：Cookie 过期、单设备登录限制 ……

核心：

1. 退出后，清空用户缓存数据，如：token、userinfo、权限相关配置 ……
2. 返回登录页。
